Role-Based Permissions

Set which roles are required to use any command. Mark certain roles as bypass roles that skip all restrictions entirely.

Prefix Per Server

Run /prefix ! and immediately use !ban, !kick and everything else. Each server has its own prefix stored persistently.

Permission System

You control who does what.

Credentials has a layered permission system built around how real servers actually work.

Bypass Role
Full access to all commands, no questions asked. Assign this to your head mods or admins.
Command Role
Require a specific role to run any command. Set different requirements per command.
Discord Perms
Base Discord permissions still apply. Layered on top of Credentials' own system.
Permission examples 
# Give a role full bypass access
/bypass add @Head Moderator

# Restrict a command to a specific role
/cmdrole set ban @Senior Mod

# Remove a command restriction
/cmdrole clear ban

# Change server prefix
/prefix !

# Now use the new prefix
!ban @user Spamming
Getting Started

Up in under three minutes.

01

Invite the bot

Click the invite link and select your server. Grant Administrator permission for full functionality, or pick individual permissions if you prefer granular control.

02

Set your prefix

Run /prefix ! (or whatever you prefer). All commands will now respond to that prefix. Default is /.

03

Configure logging

Run /logging #your-log-channel to capture all moderation actions. Recommended before you do anything else.

04

Set bypass roles

Run /bypass add @role for your senior staff so they always have full access. Add command-specific role locks with /cmdrole set.

05

Optional: tickets and welcome

Run /ticket setup and /welcome channel to activate those systems. Everything else works out of the box.
